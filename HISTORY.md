# 修改歷程

## V3.2.0

- 將版本由 `V3.1.5` 升版為 `V3.2.0`。
- 保留 `V3.1.5` 功能與版面修正內容。
- 新增 Nuitka 優化打包版，OCR 載入順序改為 RapidOCR 優先，降低打包體積與大型相依風險。
- 新增 macOS Nuitka 打包腳本與打包指南。
- 新增可直接以 pythonw 執行的 `.pyw` 版本。
- 新增正常版 `.py` 入口檔，方便直接用 Python 執行。

## V3.1.5

- 修正主視窗可自由縮放時，內部左右區塊無法跟著縮放的問題。
- 將最外層主要版面改為有欄位權重的 `grid` 配置。
- 左側 PDF 清單與右側工作區會依視窗寬度比例伸縮。
- 左側 PDF 清單新增水平捲軸。
- 左側與搬移瀏覽區的檔名欄位可隨視窗寬度伸縮，其他資訊欄保持穩定寬度。
- 依規則進版到 `V3.1.5`。

## V3.1.4

- 降低 PDF 預覽造成的記憶體佔用。
- 限制 PDF 預覽 raster 最大像素，避免高縮放或大尺寸頁面產生過大的 bitmap。
- PDF 重繪前釋放舊的 PIL/Tk 預覽圖引用。
- PDF 關閉與清除目前檔案時主動清理預覽記憶體。
- OCR 辨識後釋放預處理影像、numpy array 與裁切圖。
- 依規則進版到 `V3.1.4`。

## V3.1.3

- 修正搬移功能載入長檔名資料夾時，瀏覽區欄位被檔名撐大，導致最右側「更名 / 搬移」切換標籤被擠出視窗的問題。
- 將最右側切換標籤固定保留在右側配置。
- 固定左側與搬移瀏覽區 Treeview 欄寬，長檔名改由欄內截斷與水平捲軸承接。
- 依規則進版到 `V3.1.3`。

## V3.1.2

- 將底部前名欄位標籤由「增加前名」改為「前名調整」。
- 將後方確認按鈕由「增加前名」改為「增名確認」。
- 依規則進版到 `V3.1.2`。

## V3.1.1

- 將程式整理成優化完整碼版本。
- 新增 `APP_VERSION`，讓視窗標題由版本常數產生。
- `PDFState` 改為 `dataclass`。
- 集中時間格式、排序切換、OCR 文字更新、PDF 清理邏輯。
- 更名後自動選回新檔名。
- 搬移/刪除後正確清掉目前 PDF 預覽狀態。
- 回復搬移、回復刪除時，只有成功回復才移除歷史紀錄。
- 搬移瀏覽區倒序排序時，資料夾仍維持在檔案前面。

## V3.1

- 原始版本基礎：PDF 預覽、更名、增加前名、刪除/回復、排序、OCR 框選辨識與搬移整合。
